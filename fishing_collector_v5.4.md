## 概要

釣果データ収集ツール V5.4 は、Instagramのスクリーンショットや手動入力から釣果データを収集・管理するためのブラウザ動作型アプリケーションです。HTML単一ファイルで動作し、インターネット接続なしでも利用できます。

**主な機能：**
- Instagram スクリーンショットをAI（Claude）で解析してデータ化
- 手動でのデータ入力
- データの一覧表示・検索・フィルタ・ソート
- 重複データの検出・警告
- CSVエクスポート・インポート
- GitHub Pages を利用したマルチデバイス対応

## タブ別機能説明

### データ一覧タブ

起動時に最初に表示されるタブです。登録済みのすべての釣果データを一覧表示します。

#### 統計・操作パネル（折りたたみ式）

画面上部の **「▼ 統計・操作パネルを表示」** ボタンをタップ/クリックすると展開します。

| 統計項目 | 説明 |
|---------|------|
| 件数 | 登録済みデータの総件数 |
| 魚種数 | 登録されている魚種の種類数 |
| 最大サイズ | 登録データ中の最大サイズ（cm） |
| 最大重量 | 登録データ中の最大重量（kg） |
| Instagram | Instagramから取り込んだ件数 |
| 手動入力 | 手動で入力した件数 |

#### 操作バー

| ボタン | 機能 |
|--------|------|
| 📂 CSV読込 | CSVファイルをインポートしてデータを追加 |
| 📥 CSVで保存 | 全データをCSVファイルとしてダウンロード |
| 全削除 | 全データを削除（確認あり） |
| 🗑 選択削除 | チェックした行を削除 |

#### 絞り込み・検索

- **キーワード検索：** 魚種・ポイント・エサで絞り込み
- **魚種フィルター：** ドロップダウンで特定魚種のみ表示
- **並順リセット：** ソートを初期状態（投稿日降順）に戻す

#### データ表の列

| 列名 | 内容 |
|------|------|
| 投稿日 | 投稿日付（YY/MM/DD形式） |
| 魚種 | 魚の種類（最大7文字表示） |
| cm | サイズ（センチメートル） |
| kg | 重量（キログラム） |
| 本 | 釣果数量 |
| エサ | 使用したエサ・ルアー |
| 釣法 | 釣り方 |
| ポイント | 釣り場所 |
| 潮 | 潮回り |
| 天気 | 天気 |
| 水温 | 水温（℃） |
| 気温 | 気温（℃） |
| 風 | 風速（m/s） |
| メモ | 補足情報 |
| ソース | データ入力元 |
| 入力日時 | システムへの登録日時 |
| 複製 | この行のデータを複製して追加 |

#### 重複検出機能

同じ日付・魚種・数量・ポイントのデータが複数ある場合、**⚠️ 重複警告バナー**が表示されます。

#### 複製機能

行右端の **「複製」** ボタンをクリックすると、そのデータを元にした新しいレコードを作成できます。重量・サイズ・本数だけ変えて追加する場合に便利です。

### 解析して追加タブ

InstagramのスクリーンショットをAI（Claude）に解析させてデータを取り込む機能です。

#### 手順

**STEP 1 — Claude に依頼**

1. **「📋 プロンプトをコピー」** ボタンをクリックしてプロンプトをコピー
2. Claude のチャット画面を開く
3. スクリーンショット画像を添付する
4. コピーしたプロンプトを貼り付けて送信

**STEP 2 — JSONを貼り付け**

1. Claudeが返答したJSONをコピー
2. **「JSONを貼り付け」** エリアに貼り付ける
3. **「⚡ JSONを解析する」** ボタンをクリック

**STEP 3 — 確認して追加**

1. 解析結果が一覧表示される
2. 各件をクリックして内容を確認・編集
3. 追加したい件にチェックを入れる
4. **「＋ 選択した件をリストに追加」** をクリック

#### スクリーンショット日時の自動設定

ファイル選択ボタンでスクリーンショットファイルを選択すると、ファイル名から撮影日時を自動取得します。

対応するファイル名パターン：
- `スクリーンショット_25-3-2026_19040_...`
- `Screenshot_20260325_190400.png`
- `IMG_20260325_190400.jpg`

### 手動入力タブ

データを直接入力して登録します。

#### 入力フィールド

| フィールド | 必須 | 説明 |
|-----------|------|------|
| 投稿日付 | - | データの日付（起動時に今日の日付が入力済み） |
| 投稿時刻 | - | データの時刻 |
| 魚種 | **必須** | 魚の種類（タグボタンからも選択可） |
| サイズ cm | - | 魚のサイズ |
| 重量 kg | - | 魚の重量 |
| 本数 | - | 釣果数量 |
| エサ/ルアー | - | 使用したエサまたはルアー |
| 釣法 | - | タグボタンから選択（泳がせ・落とし込み・ジギング等） |
| ポイント | - | 釣り場所 |
| 水温 ℃ | - | 水温 |
| 気温 ℃ | - | 気温 |
| 風速 m/s | - | 風速 |
| 潮回り | - | ドロップダウンから選択 |
| 天気 | - | ドロップダウンから選択 |
| メモ | - | 自由記述 |
| 情報ソース | - | データの出典 |

**「＋ リストに追加」** ボタンで登録。登録後はフィールドがクリアされます。

## データ管理

### CSVエクスポート

1. **データ一覧タブ** を開く
2. 統計・操作パネルを展開する
3. **「📥 CSVで保存」** をクリック
4. `釣果DB_YYYYMMDD.csv` という名前でダウンロードされる

### CSVインポート

1. **データ一覧タブ** を開く
2. 統計・操作パネルを展開する
3. **「📂 CSV読込」** をクリック
4. CSVファイルを選択する
5. データが追加される（既存データは消えません）

### CSVのフォーマット

```
date,time,species,size_cm,weight_kg,count,bait,method,spot,tide,weather,temp,water_temp,wind,memo,source
2026-03-22,14:30,カンパチ,80,5.5,1,アジ,泳がせ（底物）,沖ノ島,大潮,晴れ,22,24,3,推定値,Instagram
```

### データの保存場所

データはブラウザの **localStorage** に保存されます。

| 注意点 | 内容 |
|--------|------|
| 端末ごとに独立 | PCで入力したデータはスマホに自動同期されません |
| ブラウザごとに独立 | ChromeとSafariは別々に保存されます |
| データの引き継ぎ | CSVエクスポート → 別端末でインポートして移行できます |

### GitHub Pages 経由でのデータ共有

`data.csv` を GitHub リポジトリに置くと、初回アクセス時に自動読み込みされます。

1. CSVエクスポートで `data.csv` として保存
2. GitHub の `fishing-collector` リポジトリにアップロード
3. どの端末からアクセスしても同じデータが表示される

## スマートフォン・タブレット対応

### スマホでの表示

- **統計・操作パネル** は初期状態で折りたたまれており、テーブルを広く使えます
- **「▼ 統計・操作パネルを表示」** ボタンをタップで展開/折りたたみ
- テーブルの **見出し行は固定** され、データ部分のみスクロールします
- 横スクロールで全列を確認できます

### 表示行数の目安

| デバイス | 表示行数 |
|---------|---------|
| iPhone（標準） | 約5行 |
| iPad | 約7行 |
| PC | 約15行以上 |

## GitHub Pages 公開手順

### 初回セットアップ

1. [github.com](https://github.com) でアカウントを作成
2. `fishing-collector` という名前のリポジトリを作成（Public）
3. `index.html` をアップロード
4. Settings → Pages → Branch: main → Save

公開URL：`https://[ユーザー名].github.io/fishing-collector/`

### ファイル更新手順

1. `index.html`（または `data.csv`）を修正・用意する
2. GitHub の `fishing-collector` リポジトリを開く
3. **「+」→「Upload files」** をクリック
4. ファイルをアップロード
5. **「Commit changes」** をクリック
6. 1〜2分後に公開URLに反映される
